Responsive Vs Adaptive Design

September 12, 2022 Ella Etop-Essien


NO AUDIO AVAILABLE for this article!

In this article, I will discuss about the pros and cons of responsive and adaptive design.
A responsive website uses just one layout and "responsively" adjusts to better fit the user's screen. An adaptive website has different layouts of the webpage for the differnt devices. This means that, there are multiple versions of the same website to fit someone's device as opposed to a single, static page (that reorders or resizes content) across all devices.

Responsive Design makes use of media queries to target breakpoints which scale images, wraptext & adjust layouts. HTML & CSS are used in responsive design. Not only does adaptive design make use of css media queries they also make use of JavaScript. This is done to change the site's HTML markup based on various device capabilities. Examples of websites that make use of Responsive Design are Shopify and Microsoft while, websites that make use of Adaptive Design are Apple and Amazon.

Let's analyse and compare the two designs:
The layout of a responsive design is determined by the visitor's browser window while, that of an adaptive design is determined by the backend.
The load time of an Adaptive design is faster than a responsive design.
Adaptive design is more difficult because it will require differnt layouts for differnt devices.
Responsive design is easier to maintain unlike adaptive design which will require occasional maintenance especially if there is a new design in the market.
Google recommends and rewards sites with a responsive design. SEO finds adaptive websites challenging
